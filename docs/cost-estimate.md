## 1. 課金モデル解説

各 Azure サービスの課金の仕組み・単位・単価を説明します。

### 1.1 Container Apps（Consumption プラン）

> 📖 [Container Apps の価格](https://azure.microsoft.com/pricing/details/container-apps/)

**課金単位**: vCPU 秒 + メモリ (GiB) 秒 + リクエスト数

```
月額 = (アクティブ時間 × vCPU × vCPU 単価) + (アクティブ時間 × GiB × メモリ単価)
     + (アイドル時間 × vCPU × アイドル vCPU 単価) + (アイドル時間 × GiB × アイドルメモリ単価)
     + (リクエスト数 × リクエスト単価) − 無料枠
```

| 計量 | アクティブ時 | アイドル時 |
|------|-------------|----------|
| vCPU | ¥3.65/vCPU/時 ($0.000024/秒) | ¥0.46/vCPU/時 ($0.000003/秒) |
| メモリ | ¥0.46/GiB/時 ($0.000003/秒) | ¥0.46/GiB/時 ($0.000003/秒) |
| リクエスト | ¥0.06/万件 ($0.40/100 万件) | — |

- **アクティブ vs アイドル**: vCPU 使用率 > 0.01 コア or 受信 > 1,000 bytes/秒 → アクティブ。それ以外はアイドル（単価約 1/8）
- **min_replicas = 1** の場合、トラフィックがなくてもアイドル料金が常時発生
- **無料枠（サブスクリプション単位/月）**: 180,000 vCPU 秒 + 360,000 GiB 秒 + 200 万リクエスト

### 1.2 Azure OpenAI

> 📖 [Azure OpenAI の価格](https://azure.microsoft.com/pricing/details/cognitive-services/openai-service/)

**課金単位**: 入力トークン数 + 出力トークン数（1M トークン単位）

```
月額 = (入力トークン / 1M × 入力単価) + (出力トークン / 1M × 出力単価)
```

| モデル | デプロイ種別 | 入力 (/1M tokens) | 出力 (/1M tokens) |
|--------|------------|-------------------|-------------------|
| GPT-5.4 | Global Standard | $2.50 (¥388) | $15.00 (¥2,325) |

- **基本料金なし**: デプロイを維持するだけでは課金されない。API 呼び出し時のみ
- **TPM (30K)**: 1 分あたりの最大トークン処理量。超過時はスロットリング (HTTP 429)。課金には影響しない
- **キャッシュ入力**: 同一プレフィックスのキャッシュヒット時は入力単価の 10%（$0.25/1M）

### 1.3 Azure Speech

> 📖 [Speech Service の価格](https://azure.microsoft.com/pricing/details/cognitive-services/speech-services/)

**課金単位**: 文字起こしした音声の時間（秒単位で計量、時間単位で請求）

| 方式 | 単価 (/時間) | 特徴 |
|------|-------------|------|
| Fast Transcription | $1.00 (¥155) | 同期処理。応答が速い |
| Batch Transcription | $0.18 (¥28) | 非同期。約 82% 安い |

- **実際の音声長のみ課金**: 10 分の音声 → 10 分ぶんだけ課金。API 呼び出し回数は関係なし
- **AI Services 統合**: 本構成では AIServices (Foundry) アカウント経由で利用。別途 Speech リソースは不要

### 1.4 Azure Blob Storage（Standard LRS / Hot）

> 📖 [Blob Storage の価格](https://azure.microsoft.com/pricing/details/storage/blobs/)

**課金単位**: 保存 GB + 操作回数 + データ取り出し量

```
月額 = (保存 GB × GB 単価) + (書き込み操作 / 10K × 書き込み単価) + (読み取り操作 / 10K × 読み取り単価)
```

| 計量 | 単価 (Japan East, Hot tier) |
|------|---------------------------|
| 保存データ | $0.02/GB/月 (¥3.1) |
| 書き込み操作 (PUT/LIST) | $0.055/10K 件 (¥8.5) |
| 読み取り操作 (GET) | $0.0044/10K 件 (¥0.7) |

- **日割り計算**: 保存データ量は月末の合計 GB × 日割り
- **操作課金**: REST API 呼び出し 1 回 = 1 操作。Blob アップロード = PUT 1 回
- **Private Endpoint 経由**: PE データ処理料 $0.01/GB も別途発生

### 1.5 Private Endpoint

> 📖 [Private Link の価格](https://azure.microsoft.com/pricing/details/private-link/)

**課金単位**: エンドポイント数 × 時間 + データ処理量

| 計量 | 単価 |
|------|------|
| エンドポイント時間 | $0.01/時間 (¥1.55) × PE 数 |
| データ処理 (0〜1 PB) | $0.01/GB (¥1.55) |

- **常時課金**: エンドポイントが存在する限り時間課金が発生（1 PE あたり約 $7.30/月）
- **本構成は PE ×3**: Blob + AI Services + ACR

### 1.6 Private DNS Zone

> 📖 [Azure DNS の価格](https://azure.microsoft.com/pricing/details/dns/)

| 計量 | 単価 |
|------|------|
| ゾーン (最初の 25 個) | $0.50/月 (¥78) |
| DNS クエリ | $0.40/100 万件 (¥62) |

- **本構成は 4 ゾーン**: blob / cognitiveservices / openai / azurecr

### 1.7 Container Registry（Premium SKU）

> 📖 [Container Registry の価格](https://azure.microsoft.com/pricing/details/container-registry/)

**課金単位**: SKU 日額固定料金

| 計量 | 単価 |
|------|------|
| Premium 基本料金 | $1.667/日 (¥258) → 約 $50.03/月 (¥7,755) |
| 含まれるストレージ | 500 GiB |
| 超過ストレージ | $0.003/GiB/日 |

- **日額固定**: イメージの push/pull 回数には依存しない
- **Premium が必要な理由**: Private Endpoint は Premium SKU のみ対応

### 1.8 Log Analytics

> 📖 [Azure Monitor の価格](https://azure.microsoft.com/pricing/details/monitor/)

**課金単位**: 取り込みデータ量 (GB)

| 計量 | 単価 |
|------|------|
| データ取り込み | $2.76/GB (¥428) |
| 無料枠 | 最初の 5 GB/月/課金アカウント |
| データ保持 (31 日まで) | 無料 |

## 2. リソース別コスト見積もり

### 2.1 コンピュート

**前提**: 日中 8 時間（平日 22 日）のみ利用。min_replicas = 1 のため、利用時間外もアイドル料金が発生する。

| 区分 | 時間/月 | 説明 |
|------|---------|------|
| アクティブ | 176h (8h × 22 日) | 日中業務時間帯。リクエスト処理中 |
| アイドル | 554h (730h − 176h) | 夜間・休日。レプリカは起動中だがトラフィックなし |

**Backend（0.5 vCPU / 1 GiB × 1 レプリカ）:**

| 区分 | 秒数 | vCPU コスト | メモリコスト |
|------|------|-------------|-------------|
| アクティブ (176h) | 633,600s | 633,600 × 0.5 × $0.000024 = **$7.60** | 633,600 × 1 × $0.000003 = **$1.90** |
| アイドル (554h) | 1,994,400s | 1,994,400 × 0.5 × $0.000003 = **$2.99** | 1,994,400 × 1 × $0.000003 = **$5.98** |
| **小計** | | **$10.59** | **$7.88** |

Backend 合計: **$18.47（¥2,863）**

**Frontend（0.5 vCPU / 1 GiB × 1 レプリカ）:**

Frontend も同じスペック・同じ利用パターンのため: **$18.47（¥2,863）**

**無料枠控除（サブスクリプション単位/月）:**

| 計量 | 使用量合計 | 無料枠 | 概算控除額 |
|------|-----------|--------|-----------|
| vCPU 秒 (アクティブ) | 633,600 | 180,000 | −$2.16 |
| GiB 秒 (アクティブ) | 1,267,200 | 360,000 | −$0.54 |

**Container Apps 月額合計: 約 $34.24（¥5,307）**

> **参考**: アクティブ時間の比率（176h / 730h = 24%）で見ると、常時フルアクティブ想定（¥10,200）の約半額。アイドル単価が vCPU についてはアクティブの 1/8 であるため大幅に安くなる。

> **さらにコスト削減するには**: `min_replicas = 0` に設定すると夜間・休日はレプリカが 0 になり、アイドル料金が完全になくなる（コールドスタート 数秒が許容できる場合）。その場合の月額は約 **$19.24（¥2,982）**。

## 4. コスト最適化ヒント

### 4.1 すぐにできること

| 施策 | 効果 | 方法 |
|------|------|------|
| **Container Apps のスケールダウン** | コンピュート費用削減 | `min_replicas = 0` に設定（コールドスタートが許容できる場合） |
| **GPT TPM の適正化** | 不要なプロビジョンコスト回避 | 実際の利用量に合わせて `openai_deployment_capacity` を調整 |
| **Log Analytics の取り込み制限** | ログコスト削減 | ログレベルを WARNING に上げる、不要なログを除外 |
| **未使用イメージの削除** | ACR ストレージ削減 | `az acr repository delete` で古いタグを定期削除 |

### 4.2 中長期的な最適化

| 施策 | 効果 | 備考 |
|------|------|------|
| **Reserved Instance** | Container Apps コスト 30〜50% 削減 | 1 年 / 3 年のコミットメント |
| **Private Endpoint の集約** | ネットワークコスト削減 | 現在 PE ×3。閉域要件が緩和できれば削減可能 |
| **Batch Transcription** | 長時間音声のコスト削減 | Fast Transcription より約 82% 安価（$0.18/時 vs $1.00/時）。非同期 |
| **モデルの見直し** | AI コスト大幅削減 | 議事録生成に GPT-4o-mini 等の軽量モデルを検討 |
| **リージョン選択** | 価格差の活用 | East US 等の安価なリージョンでの AI サービス利用 |

### 4.3 コスト監視の設定

```bash

az consumption budget create \
  --budget-name meeting-minutes-monthly \
```
